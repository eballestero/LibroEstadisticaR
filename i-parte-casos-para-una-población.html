2.1 Simulación de Intervalos de Confianza

2.1.1 Método de simulación: Ejemplo 1

El siguiente ejemplo simula el funcionamiento de una IC para una distribución normal. Consideramos una población de 106 valores de una distribución normal de parámetros 𝜇 = 1.5 y 𝜎 = 1 :

set.seed(1012)
mu = 1.5; sigma = 1; alpha = 0.05
Poblacion = rnorm(10^6,mu,sigma)

Notas: Si desea indagar más sobre el uso del set.seed

Seguidamente se generarán 100 muestras aleatorias simples de tamaño 50, tomadas de la población que se generó en el chunk anterior.

¿Qué se desea hacer con este muestreo?

Generar un IC de 100(1-𝛼)% para 𝜇 para cada muestra. El siguiente código crea una función llamada ICZ que recibe los siguientes parámetros: x: una muestra 𝜎: desviación estándar 𝛼: nivel de significancia

ICZ <- function(x, sigma, alpha){
 c(mean(x) - qnorm(1-alpha/2)*sigma/sqrt(length(x)),
 mean(x) + qnorm(1-alpha/2)*sigma/sqrt(length(x)))}

Usando la función replicate de R generamos las muestras y los intervalos de confianza correspondientes usando la función anterior:

2.1.2 Método de simulación: Ejemplo 2

El segudo método muestra como realizar simulaciones de Intervalos de Confianza de manera gráfica, usando el paquete PASWR2. Los requerimientos para usar la función cisim, son los siguientes:

  • Samples: Número de muestras deseadas.
  • n: Tamaño de cada muestra.
  • Parameter: El valor del parámetro poblacional. Esto está sujeto al Type (media, varianza o porporción). Para el caso de type = “mean” o type = “Var” el valor puede ser cualquiera, pero si se usa proporción con type = “Pi”, entonces el valor del parámetro debe ser un valor entre 0 y 1.
  • Sigma: Desviación estándar de la población, En el caso de usar type =“Pi”, no se requiere indica este valor.
  • conf.level: Intervalo de confianza es un valor entre 0 y 1. Si es de 90%, debe escribirlo como 0.90. Cuando este parámetro se omite, R toma por defecto el IC de 95%.
  • type: Es una variable tipo character string,que ademite los siguiente valores: “Mean” para la media, “Var” para la varianza o “Pi” para proporción (de forma alternativa, puede usar solo la primera letra en casad caso). Si no se coloca type en la instrucción, se asume por defecto type = “Mean”.

Ejemplo Simulación de 100 muestras de tamaño 30, con media 100 y desviación estándar de 10. El IC es de 90%

#install.packages("PASWR2", dependencies = T)
cisim(samples = 100, n = 30, parameter = 100, sigma = 10, conf.level =
0.90)

2.1.3 Experimento con la “Confianza”

Vamos a comprobar con un experimento qué papel juega la “confianza” en los intervalos de confianza. Vamos a generar al azar una Población de 10 000 000 (107 ) “individuos” con distribución normal estándard. Se va a tomar 200 muestras aleatorias simples de tamaño 50 de esta población y se calculará el intervalo de confianza para la media poblacional usando dicha fórmula. Finalmente, la idea es contar cuántos de estos intervalos de confianza contienen la media poblacional. Se fijará la semilla de aleatoriedad para que el experimento sea reproducible.

# Se genera la población de valores
set.seed(2020)
valores.poblacion <- rnorm(10^7)

# Se calcula la media poblacional
mu <- mean(valores.poblacion)

# Se toman 200 muestras usando la función "replicate" de R, que permite ejecutar una misma función las veces que se le indique
muestras <- replicate(200, sample(valores.poblacion, 50, replace = TRUE))
str(muestras)
##  num [1:50, 1:200] 0.59 -0.192 0.346 -0.524 -0.283 ...
## num [1:50, 1:200] 0.59 -0.192 0.346 -0.524 -0.283 ...

# Note que la variable muestras es una matriz de 50 filas por 200 columnas, donde cada una de las columnas representa una muestra u observación.

# Usando t.test para cada una de las muestras, se calculará un IC de 95%, para posteriormente contar los aciertos, es decir, cuántos de ellos continenen a la media poblacional.

# Se define la función IC.t que nos da el IC para la media dada para una muestra
IC.t <- function(X, confianza = 0.95){
  t.test(X, conf.level = confianza)$conf.int
}

ICs <- apply(muestras, FUN = IC.t, MARGIN = 2)

# Conteo de aciertos
Aciertos <- length(which((mu >= ICs[1,]) & (mu <= ICs[2,])))
Aciertos
## [1] 195
## [1] 195

Note que se ha acertado en 195 veces. o sea. un 97.5% de las veces, lo cual es una buena aproximación del valor 95% que era el esperado. Seguidamente, se puede visualizar gráficamente estos intervalos:

plot(1, type = "n", xlim = c(-0.8, 0.8), ylim = c(0, 200), xlab = "Valores", ylab = "Repeticiones", main = "")

  }

}

2.2 Intervalos de confianza para el parámeto p

Aunque en el curso sólo de abordó un método, en esta sección se estudiarán 3 métodos para estimar el parámetro 𝑝.

2.2.1 Método “exacto” o de Clopper-Pearson

Consideremos lo siguiente:

\[ \begin{equation*} \begin{aligned} & X \text{ una v.a. Bernoulli con } p \text{ desconocido} \\ & X_1, \ldots, X_n \text{ son una muestra aleatoria simple (m.a.s.) de } X, \\ & \text{ con un número de éxitos } x. \\ & \text{ Por lo tanto, la frecuencia relativa de éxitos es } \hat{p}_X = \frac{x}{n}. \end{aligned} \end{equation*} \] En este caso, la distribución de la variable \(Y\) = “número de éxitos en la muestra” es binomial de parametros \(n \text{ y } p \text{,} Y \text{ es } B(n,p)\)

Definición.

Un intervalo de confianza \((p_0 \;\text{, } p_1)\) del \((1 - a)100%\) nivel de confianza para \(p\) de una población \(X\) de Bernoulli se obtiene encontrando el \(p_0\) más grande y el \(p_1\) más pequeño tales que:

\[ \sum_{k=x}^{n} \left( \frac{n}{k} \right) \cdot p_0^k \; \cdot (1-p_0)^{n-k} \leq \frac{a}{2}, \; \; \sum_{k=0}^{x} \left( \frac{n}{k} \right) \; \cdot p_1^k \; \cdot (1-p_1)^{n-k} \; \leq \frac{a}{2} \] Para hallar el IC para la proporción usando este método, se usará la funión binom.exact que se encuentra en el paquete epitools que se cagó al inicio: binom.exact(x,n,conf.level)

x: número de éxitos n: tamaño de la muestra

conf.level nivel de confianza correspondiente a \(1 − a\)

Se trabajará con la base de datos iris, donde se hallará un IC para las proporciones de la especie “setosa” a partir de una muestra de 60, con una confianza de 95%:

2.2.2 Método de Wilson

Definición: En estas condiciones, un intervalo de confianza del \((1-a) \; \cdot \; 100\)% I.C. para \(p\) (donde) \(\left (\hat{p}_X = 1 - \hat{p}_X \right)\) es:

\[ \left ( \frac{\hat{p}_X + \frac{z_1^2 -a/2}{2n}-z_{1-a/2} \; \sqrt{\frac{\hat{p}_X \cdot \hat{q}_X}{n} + \frac{z^2_{1-a/2}}{4n^2}}} {1 + \frac{z_{1-a/2}}{n}} , \frac{\hat{p}_X + \frac{z_{1-a/2}^2}{2n} + z_{1-a/2} \sqrt{\frac{\hat{p}_X \cdot \hat{q}_X}{n} + \frac{z_{1-a/2}^2}{4n^2}}} {1 + \frac{z_{1-a/2}^2}{n}}\right) \cdot \] Para hallar un intervalo de confianza para la proporción poblacional en R según el método de Wilson, hay que usar la función binom.wilson del mismo paquete epitools.

Se usa la función binom.wilson del paquete epitools: binom.wilson(x,n,conf.level)

Los parámetros requeridos son los mismo que el caso del método exacto

binom.wilson(numero.flores.setosa, 60, conf.level = 0.95) ## x n proportion lower upper conf.level ## TRUE 21 60 0.35 0.2416777 0.4763738 0.95

2.2.3 Método de Laplace

Supongamos que la muestra aleatoria simple es considerablemente más grande que la usada en el método de Wilson y que, además, la proporción muestral de éxitos $ _X $ está alejada de 0 y de 1.

  • O sea, \(n \geq 100\) y que \(n\hat{p}_X \;\geq \; 10\) y \(n(1- \hat{p}_X) \geq 10\).
  • En este caso, podemos usar la fórmula de Laplace:

\[ \hat{p}_X \pm z_{1 - a/2} \cdot \sqrt{\frac{\hat{p}_X(1-\hat{p}_X)}{n}} \] Se usa la función binom.approx del paquete epitools:

binom.approx(x,n,conf.level) Los parámetros requeridos son los mismo que el caso del método exacto. Este método es que estudiamos en el curso, por cual debe ser considerado siempre que se utilicen muestras grandes.

binom.approx(numero.flores.setosa, 60, conf.level = 0.95) ## x n proportion lower upper conf.level ## TRUE 21 60 0.35 0.2293123 0.4706877 0.95

2.3 Intervalo de confianza para la varianza de una población normal

Consideremos la siguiente situación:

  • Consideramos una \(X\) una v.a normal con \(\mu\) y \(\sigma\) desconocidas.

  • Sea \(X_1 \dots, X_n\) una m.a.s de X y varianza Muestral \(\hat{S}^{2}_X\)

En estas condiciones tenemos lo siguiente:

Teorema

La variable aleatyoria \(\frac{(n-1)\hat{S}^2_X}{\sigma^2}\) se distribuye según una distribución \(\chi^2_{n-1}\)

Teorema

En las condiciones anteriores, un intervalo de confianza del \((1-a) \cdot 100%\) para la varianza de \(\sigma^2\) de la población \(X\) es

\[ \left(\frac{(n-1)\hat{S}^2_X}{\chi^2_{n-1 \; ,\; 1-\frac{a}{2}}} \; , \; \frac{(n-1)\hat{S}^2_X}{\chi^2_{n-1 \; ,\; \frac{a}{2}}}\right), \] dónde \(\chi^2_{v,q}\) es el q-cuantil de la distribución \(\chi^2{v}\)

Ejemplo

Un algoritmo probabilístico depende de la semilla de aleatorización que se genera en cada paso. Para saber si la semilla influye mucho en el resultado se ejecuta el algoritmo varias veces hasta obtener un resultado similar y se estudia la varianza de su tiempo de ejecución. Queremos ver si la desviación típica cumple que 𝜎 ≤30. Se supone que la distribución del tiempo de ejecución del algoritmo es aproximadamente normal. Se realizan 30 ejecuciones del algoritmo de las que se mide el tiempo de ejecución. Los resultados son:

Se requiere calcular un IC de 95% para \(\sigma^2\)

# Varianza muestral 
n <- 30 s.tiempo <- var(tiempo)

# Calcular los valores de la distribución o cuantiles

alpha <- 0.05 chi2.izq <- qchisq(1-alpha/2, n-1) chi2.der <- qchisq(alpha/2, n-1) 

# IC para la varianza: 

ICvar <- c((n-1)*s.tiempo/chi2.izq, (n-1)*s.tiempo/chi2.der)
